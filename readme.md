# 银行周报
我计划使用git action + Go语言 + LLM + Github pages 来搭建一个能自动化生成银行周报的项目，其内容主要有。
* 1. 利用Go语言来抓去各大银行的Research文章。
* 2. 利用LLM来将各大银行的Research文章进行总结分析。
* 3. 每月会对上述总结出来的各大银行的Research文章进行分析，生成月报。
* 4. 利用Go语言来发送周报和月报到指定的邮箱，并构建一个Github pages web页面来展示周报和月报。

### 1. 技术栈
* 1. Go语言
* 2. LLM
* 3. Git Action
* 4. Html + CSS + JS

### 2. 流程分析。

* 1. 利用Go语言来抓取给出的银行和金融机构Research文章，即给出指定银行的Research文章的url，然后利用Go语言来抓取各大银行的Research文章，或者Global review文章，同时要聚焦于亚洲区域（这一部分可以修改）。
* 2. 利用LLM来将各大银行的Research文章进行总结分析，尤其要注重各大银行和金融机构对整个事件的看法，包括对未来一年或者半年走势的推测、对行业形势研究、局势安全、进出口、PPI、CPI等各种看法。
尤其要注意这一个部分中，LLM不允许有自己的看法，仅仅能作为对Research或Review文章的忠实总结概括，
LLM需要以Markdown格式来进行输出，对LLM的输出不限制输出长度，并非意味着只是简单的概括总结，还要将其中的推理（如果有的话就要体现出来，如果没有则不要书写），和前因后果体现出来，并以{中文}（这里设置Github变量，可以在github仓库的变量中调整）输出。{LLM名字和LLM的URl}都要在Github中以Github 变量的形式体现，以防止泄漏API，同时保证可以更换LLM，
另外给出的Research文章的url，需要在Github中以Github 变量的形式体现，这样可以随时更换不同金融机构的研报。
* 3. 每月会对上述总结出来的各大银行的Research文章进行分析，生成月报。
这里需要注意利用LLM来分析各金融机构，尤其是对相同事件的看法（也许相同也许不同）。
分类表格列举不同机构的看法（如果需要）。
* 4. 利用Go语言来发送周报月报到指定的邮箱（查看Github action 是否可以实现该功能，如果不方便实现，就忽略这个步骤）
并构建一个Github pages 用于展示所生成过的文章，以便随时访问。
所需要构建的Github pages 基本样式可以参考/Ref文件夹下的目录，仅参考功能，但是Css、Js、Html的版面样式需要进行改变。
